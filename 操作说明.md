# IEC104 协议测试工具操作说明

## 1. 工具概述

IEC104 协议测试工具是一个基于 openmuc.j60870 的 Java Swing 桌面程序，用于 IEC 60870-5-104 协议的学习、联调、报文观察和设备模拟测试。

工具提供两个工作模式：

- 客户端模式：作为 IEC104 主站/客户端，连接远端 IEC104 服务端，支持多连接管理、总召唤、时钟同步、遥控、设点、读命令、测试命令、复位进程、报文监控和数据分类显示。
- 服务端模式：作为 IEC104 从站/服务端，监听指定地址和端口，模拟数据点，响应客户端连接、总召唤和控制命令，并支持突发上送和报文监控。

主窗口包含：

- 顶部菜单：文件、帮助。
- 中间区域：客户端模式、服务端模式两个页签。
- 底部状态栏：显示当前状态、报文数和当前模式。

## 2. 运行环境

### 2.1 基本要求

- 操作系统：Windows、Linux 或 macOS，推荐 Windows。
- Java：JDK 1.8 及以上。
- Maven：Maven 3.x，源码构建时需要。
- 默认 IEC104 端口：2404。

### 2.2 打包要求

如需使用 `build-package.ps1` 生成免安装应用镜像或 MSI 安装包，需要：

- JDK 14 及以上，推荐 JDK 17 及以上。
- 正确配置 `JAVA_HOME`。
- 生成 MSI 安装包时，Windows 环境通常还需要 WiX Toolset。

## 3. 启动方式

### 3.1 在 IDE 中启动

1. 使用 IntelliJ IDEA 或其他 Java IDE 打开项目目录。
2. 等待 Maven 依赖加载完成。
3. 运行主类：

```bash
com.iec104tester.Main
```

4. 程序启动后显示主窗口，默认进入“客户端模式”。

### 3.2 使用 Maven 启动

在项目根目录执行：

```bash
mvn compile exec:java -Dexec.mainClass="com.iec104tester.Main"
```

### 3.3 打包为 JAR 后启动

在项目根目录执行：

```bash
mvn clean package
java -jar target/iec104-tester-1.0.0.jar
```

项目使用 `maven-shade-plugin` 生成包含依赖的 fat JAR，通常不需要额外配置 classpath。

### 3.4 使用 PowerShell 打包脚本

在 Windows PowerShell 中执行：

```powershell
.\build-package.ps1
```

脚本会尝试完成以下步骤：

1. 使用 Maven 编译并生成 fat JAR。
2. 使用 `jpackage` 生成应用镜像。
3. 尝试生成 MSI 安装包。

如果 MSI 生成失败，但应用镜像已生成，可以直接运行应用镜像目录中的 `.exe` 文件。

## 4. 主菜单操作

### 4.1 文件菜单

文件菜单包含以下功能：

| 菜单项 | 功能说明 |
| --- | --- |
| 新建会话 | 清空当前模式下的报文记录。客户端模式下清空当前选中连接的报文；服务端模式下清空服务端报文。 |
| 打开报文文件 | 打开已保存的 `.jsonl` 报文文件，并加载到当前模式的报文列表中。 |

| 导出CSV | 将当前模式下的报文记录导出为 `.csv` 文件，便于使用 Excel 或其他工具分析。 |

### 4.2 帮助菜单

点击“帮助” -> “关于”，可以查看工具版本和基本说明。

## 5. 客户端模式操作

客户端模式用于连接外部 IEC104 服务端，也可以连接本工具服务端模式启动的本地服务。

### 5.1 界面组成

客户端模式左侧是“连接列表”，右侧是当前连接的操作和数据区域。

左侧按钮包括：

| 按钮 | 功能说明 |
| --- | --- |
| 新建 | 新建一个客户端连接配置。 |
| 连接 | 使用当前连接配置连接 IEC104 服务端。 |
| 断开 | 断开当前连接。 |
| 设置 | 修改当前连接的 IP、端口、公共地址、IOA 范围、超时和协议参数。 |
| 删除 | 删除当前连接；如连接正在使用，会提示是否先断开。 |
| 保存场景 | 将当前所有客户端连接配置保存为 `.iec104` 场景文件。 |
| 加载场景 | 从 `.iec104` 场景文件恢复客户端连接列表。 |

右侧页签包括：

| 页签 | 功能说明 |
| --- | --- |
| 报文监控 | 显示当前连接收发的 IEC104 报文，并可查看报文详情。 |
| 遥信 | 显示接收到的遥信类数据。 |
| 遥测 | 显示接收到的遥测和电度类数据。 |
| 遥调 | 显示接收到的设点类数据。 |
| 遥控 | 显示接收到的遥控类数据。 |

底部命令按钮包括：

| 按钮 | 功能说明 |
| --- | --- |
| 总召唤 | 发送总召唤命令。 |
| 电度召唤 | 发送电度召唤命令。 |
| 时钟同步 | 发送时钟同步命令。 |
| 单点遥控 | 发送单点遥控命令。 |
| 双点遥控 | 发送双点遥控命令。 |
| 设点(浮点) | 发送短浮点设点命令。 |
| 设点(标度) | 发送标度值设点命令。 |
| 读命令 | 按信息对象地址读取数据。 |
| 测试命令 | 发送测试命令。 |
| 复位进程 | 发送复位进程命令。 |

未连接时，底部命令按钮不可用；连接成功后自动启用。

### 5.2 新建连接

1. 进入“客户端模式”。
2. 点击左侧“新建”。
3. 连接列表中会出现一个新连接，例如“连接 1”。
4. 选中新连接后，点击“设置”配置连接参数。

### 5.3 配置连接参数

点击“设置”后进入“连接配置”窗口。

基本设置：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 主机地址 | 127.0.0.1 | IEC104 服务端 IP 地址。 |
| 端口 | 2404 | IEC104 服务端端口。 |
| 公共地址 | 1 | ASDU 公共地址，需与对端一致。 |
| 源发地址 | 0 | Originator Address，通常保持默认。 |

IOA 范围配置：

| 数据类型 | 默认起始地址 | 默认数量 | 说明 |
| --- | ---: | ---: | --- |
| 遥信 | 0x0001 | 0 | 数量为 0 表示不限制范围，显示所有接收到的数据。 |
| 遥测 | 0x4001 | 0 | 数量为 0 表示不限制范围。 |
| 遥控 | 0x6001 | 0 | 数量为 0 表示不限制范围。 |
| 遥调 | 0x6201 | 0 | 数量为 0 表示不限制范围。 |

超时参数：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| 连接超时(t0) | 30000 ms | 建立 TCP/IEC104 连接的超时时间。 |
| 报文分段超时 | 5000 ms | 报文分段接收超时时间。 |
| t1 | 15000 ms | 发送或测试 APDU 的确认超时。 |
| t2 | 10000 ms | 未确认 I 帧的确认等待时间。 |
| t3 | 20000 ms | 空闲测试帧周期。 |

协议参数：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| k | 12 | 最大未确认 I 帧数。 |
| w | 8 | 最大未确认接收帧数。 |

字段长度：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| 传送原因字段长度 | 2 | COT 字段长度，支持 1 或 2。 |
| 公共地址字段长度 | 2 | CA 字段长度，支持 1 或 2。 |
| IOA字段长度 | 3 | IOA 字段长度，支持 1、2 或 3。 |

配置完成后点击“确定”保存。

### 5.4 连接服务端

1. 在左侧连接列表中选择目标连接。
2. 确认配置中的主机地址、端口、公共地址与对端一致。
3. 点击“连接”。
4. 连接成功后：
   - 状态栏显示当前连接状态。
   - 程序会自动发送一次总召唤。
   - “报文监控”页签开始显示收发报文。
   - 底部命令按钮变为可用。

如连接失败，请检查：

- 目标 IP 和端口是否正确。
- 对端服务是否已启动。
- 本机防火墙是否允许访问。
- 公共地址、字段长度等协议参数是否与对端一致。

### 5.5 查看报文

1. 打开当前连接右侧的“报文监控”页签。
2. 左侧表格显示报文列表。
3. 点击任意报文，右侧详情区域显示 ASDU 类型、传送原因、公共地址、信息对象、原始 HEX 等解析信息。
4. 底部状态栏“报文数”会实时更新。

### 5.6 查看遥信、遥测、遥调、遥控数据

连接后，工具会根据收到的 ASDU 类型将数据分到不同页签：

- 遥信：单点、双点、步位、位串等状态类数据。
- 遥测：归一化值、标度值、短浮点值、电度等测量类数据。
- 遥调：设点命令相关数据。
- 遥控：单点遥控、双点遥控、调节步命令等控制类数据。

数据表通常包含 IOA、类型、值、品质、传送原因和更新时间等信息。

### 5.7 发送总召唤

1. 确认连接状态为已连接。
2. 点击“总召唤”。
3. 输入公共地址和召唤限定词。
4. 默认召唤限定词为 `20`。
5. 点击“确定”发送。
6. 在“报文监控”中查看发送报文和对端响应。

### 5.8 发送电度召唤

1. 点击“电度召唤”。
2. 输入公共地址、请求类型和冻结类型。
3. 点击“确定”发送。
4. 查看对端返回的电度数据。

### 5.9 发送时钟同步

1. 点击“时钟同步”。
2. 输入公共地址。
3. 点击“确定”发送当前系统时间。
4. 在“报文监控”中确认命令发送及对端响应。

### 5.10 发送遥控命令

单点遥控：

1. 点击“单点遥控”。
2. 输入公共地址和信息对象地址。
3. 勾选或取消“合闸(ON)”。
4. 设置限定词。
5. 如需要选择-执行流程，勾选“选择(先选择后执行)”。
6. 点击“确定”发送。

双点遥控：

1. 点击“双点遥控”。
2. 输入公共地址和信息对象地址。
3. 选择命令状态：`OFF`、`ON`、`NOT_PERMITTED_A` 或 `NOT_PERMITTED_B`。
4. 设置限定词。
5. 如需要选择-执行流程，勾选“选择(先选择后执行)”。
6. 点击“确定”发送。

注意：遥控命令应仅在测试环境或确认安全的仿真环境中使用，避免误操作真实一次设备或生产系统。

### 5.11 发送设点命令

设点(浮点)：

1. 点击“设点(浮点)”。
2. 输入公共地址、信息对象地址、浮点设点值和限定词 QL。
3. 如需要选择-执行流程，勾选“选择(先选择后执行)”。
4. 点击“确定”发送。

设点(标度)：

1. 点击“设点(标度)”。
2. 输入公共地址、信息对象地址、标度值和限定词 QL。
3. 标度值范围为 `-32768` 至 `32767`。
4. 如需要选择-执行流程，勾选“选择(先选择后执行)”。
5. 点击“确定”发送。

### 5.12 发送读命令、测试命令和复位进程

读命令：

1. 点击“读命令”。
2. 输入公共地址和信息对象地址。
3. 点击“确定”发送。

测试命令：

1. 点击“测试命令”。
2. 输入公共地址。
3. 点击“确定”发送。

复位进程：

1. 点击“复位进程”。
2. 输入公共地址和复位限定词。
3. 点击“确定”发送。

### 5.13 保存和加载客户端场景

保存场景：

1. 配置好一个或多个客户端连接。
2. 点击左侧“保存场景”。
3. 选择保存路径。
4. 工具会保存为 `.iec104` 文件。

加载场景：

1. 点击左侧“加载场景”。
2. 选择已保存的 `.iec104` 文件。
3. 如果当前存在已连接会话，工具会提示确认是否断开并清除现有连接。
4. 加载完成后，连接列表会按文件内容重建。

## 6. 服务端模式操作

服务端模式用于模拟 IEC104 从站，供其他主站客户端或本工具客户端模式连接测试。

### 6.1 界面组成

顶部控制区包括：

| 控件 | 功能说明 |
| --- | --- |
| 绑定地址 | 服务端监听地址，默认 `0.0.0.0`。 |
| 端口 | 服务端监听端口，默认 `2404`。 |
| 高级设置 | 配置最大连接数、公共地址、数据点类型、超时、协议参数、字段长度和突发上送。 |
| 启动 | 启动 IEC104 服务端监听。 |
| 停止 | 停止服务端监听。 |
| 保存配置 | 保存服务端配置为 `.iec104` 文件。 |
| 加载配置 | 从 `.iec104` 文件加载服务端配置。 |

下方页签包括：

| 页签 | 功能说明 |
| --- | --- |
| 数据点管理 | 添加、批量生成、编辑、删除、修改模拟数据点。 |
| 报文监控 | 查看服务端收发报文及解析详情。 |
| 连接列表 | 查看已连接客户端，并可断开选中连接。 |

### 6.2 配置服务端基础参数

1. 进入“服务端模式”。
2. 在顶部输入绑定地址和端口。
3. 默认绑定地址为 `0.0.0.0`，表示监听本机所有网卡。
4. 默认端口为 `2404`。
5. 如果端口已被占用，可改为其他端口，例如 `2405`。

常用绑定地址示例：

| 地址 | 说明 |
| --- | --- |
| 0.0.0.0 | 监听所有网卡，适合被其他机器访问。 |
| 127.0.0.1 | 仅允许本机访问，适合本机自测。 |
| 本机实际 IP | 只监听指定网卡地址。 |

### 6.3 高级设置

点击“高级设置”打开服务端配置窗口。

基本设置：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| 绑定地址 | 0.0.0.0 | 服务监听地址。 |
| 端口 | 2404 | 服务监听端口。 |
| 最大连接数 | 100 | 允许同时连接的客户端数量。 |
| 公共地址 | 1 | ASDU 公共地址。 |

数据点配置：

| 数据类型 | 默认起始地址 | 默认数量 | 说明 |
| --- | ---: | ---: | --- |
| M_SP_NA_1 - 单点遥信 | 0x0001 | 20 | 默认生成 20 个单点遥信。 |
| M_DP_NA_1 - 双点遥信 | 0x2001 | 10 | 默认生成 10 个双点遥信。 |
| M_SP_TB_1 - 带时标单点 | 0x3001 | 0 | 默认不生成。 |
| M_ME_NA_1 - 归一化遥测 | 0x4001 | 0 | 默认不生成。 |
| M_ME_NB_1 - 标度化遥测 | 0x4001 | 0 | 默认不生成。 |
| M_ME_NC_1 - 短浮点遥测 | 0x4001 | 50 | 默认生成 50 个短浮点遥测。 |
| M_IT_NA_1 - 电度 | 0x4033 | 10 | 默认生成 10 个电度点。 |

可以点击：

- 默认预设：恢复默认地址和数量。
- 全部清零：将所有类型数量置为 0。

其他参数：

| 参数 | 默认值 | 说明 |
| --- | ---: | --- |
| t1 | 15000 ms | 未确认超时。 |
| t2 | 10000 ms | 确认超时。 |
| t3 | 20000 ms | 空闲测试周期。 |
| k | 12 | 最大未确认 I 帧数。 |
| w | 8 | 最大未确认接收帧数。 |
| COT长度 | 2 | 传送原因字段长度。 |
| CA长度 | 2 | 公共地址字段长度。 |
| IOA长度 | 3 | 信息对象地址字段长度。 |
| 启用突发上送 | 默认启用 | 定时主动上送模拟数据。 |
| 上送周期 | 1 秒 | 突发上送间隔。 |

配置完成后点击“确定”。

### 6.4 启动服务端

1. 设置绑定地址和端口。
2. 根据需要点击“高级设置”调整公共地址、数据点和协议参数。
3. 点击“启动”。
4. 启动成功后：
   - “启动”按钮不可用。
   - “停止”按钮可用。
   - 绑定地址、端口、高级设置和加载配置不可修改。
   - 状态栏显示“运行中”。

如果数据点管理中没有任何数据点，启动时会根据服务端配置自动生成默认数据点。

### 6.5 停止服务端

1. 点击“停止”。
2. 服务端停止监听并断开连接。
3. 状态恢复为“已停止”。
4. 可重新修改配置或加载配置。

### 6.6 管理数据点

进入“数据点管理”页签。

添加数据点：

1. 点击“添加”。
2. 填写地址、名称、类型、当前值、品质、死区和标度系数。
3. 点击“确定”。

可选数据类型包括：

- M_SP_NA_1 - 单点信息
- M_DP_NA_1 - 双点信息
- M_ME_NA_1 - 归一化测量值
- M_ME_NB_1 - 标度化测量值
- M_ME_NC_1 - 短浮点测量值
- M_IT_NA_1 - 电度
- M_SP_TB_1 - 带时标单点信息

批量生成数据点：

1. 点击“批量生成”。
2. 为每种数据类型设置起始地址、数量和名称前缀。
3. 可点击“默认预设”快速填充常用地址分区。
4. 点击“生成”。

编辑数据点：

1. 在数据点表格中选中一行。
2. 点击“编辑”。
3. 修改参数后点击“确定”。

删除数据点：

1. 在数据点表格中选中一行。
2. 点击“删除”。

修改值：

1. 在数据点表格中选中一行。
2. 点击“修改值”。
3. 输入新数值。
4. 点击“确定”。

全部清空：

1. 点击“全部清空”。
2. 确认后删除所有数据点。

### 6.7 查看服务端报文

1. 进入“报文监控”页签。
2. 表格中显示服务端接收和发送的报文。
3. 点击任意报文查看详细解析。
4. 可通过主菜单“文件” -> “保存报文”或“导出CSV”保存当前服务端报文记录。

### 6.8 管理客户端连接

1. 进入“连接列表”页签。
2. 表格显示当前连接的客户端 ID 和客户端地址。
3. 选中某个客户端。
4. 点击“断开选中连接”，可主动断开该客户端。

### 6.9 保存和加载服务端配置

保存配置：

1. 设置好绑定地址、端口和高级配置。
2. 点击“保存配置”。
3. 选择保存位置。
4. 工具会保存为 `.iec104` 文件。

加载配置：

1. 确保服务端未启动。
2. 点击“加载配置”。
3. 选择包含服务端配置的 `.iec104` 文件。
4. 加载完成后，界面会刷新为文件中的配置。

## 7. 报文保存、打开和导出

### 7.1 保存报文

1. 在客户端模式或服务端模式中完成测试。
2. 点击“文件” -> “保存报文”。
3. 选择保存路径。
4. 默认文件名格式为：

```text
iec104_capture_yyyyMMdd_HHmmss.jsonl
```

5. 保存成功后，报文以 UTF-8 编码的 JSON Lines 格式存储。

### 7.2 打开报文文件

1. 点击“文件” -> “打开报文文件”。
2. 选择 `.jsonl` 文件。
3. 工具会加载文件中的报文记录到当前模式的报文列表。

### 7.3 导出 CSV

1. 点击“文件” -> “导出CSV”。
2. 选择保存路径。
3. 工具会导出 `.csv` 文件。

CSV 字段包括：

- 时间
- 方向
- 帧类型
- ASDU类型
- 类型ID
- 传送原因
- 公共地址
- 源发地址
- 信息对象
- 摘要
- 原始HEX

CSV 文件包含 UTF-8 BOM，可直接用 Excel 打开。

## 8. 典型测试流程

### 8.1 本机自测流程

可以使用本工具同时模拟服务端和客户端。

1. 启动工具。
2. 切换到“服务端模式”。
3. 将绑定地址设为 `127.0.0.1` 或 `0.0.0.0`。
4. 端口保持 `2404`。
5. 点击“启动”。
6. 切换到“客户端模式”。
7. 选择默认连接，点击“设置”。
8. 主机地址填写 `127.0.0.1`，端口填写 `2404`，公共地址填写 `1`。
9. 点击“确定”。
10. 点击“连接”。
11. 连接成功后，客户端会自动发送总召唤。
12. 在客户端“报文监控”“遥信”“遥测”页签查看返回数据。
13. 在服务端“报文监控”“连接列表”页签查看客户端连接和收发报文。

### 8.2 连接外部 IEC104 服务端

1. 确认外部服务端 IP、端口、公共地址和字段长度。
2. 进入“客户端模式”。
3. 新建连接或选择已有连接。
4. 点击“设置”，填写对端参数。
5. 点击“连接”。
6. 连接成功后发送总召唤、读命令或其他测试命令。
7. 观察报文和数据页签。
8. 测试结束后点击“断开”。
9. 如需留存测试结果，点击“文件” -> “保存报文”或“导出CSV”。

### 8.3 模拟服务端供外部主站测试

1. 进入“服务端模式”。
2. 绑定地址使用 `0.0.0.0` 或本机实际网卡 IP。
3. 端口使用 `2404` 或双方约定端口。
4. 点击“高级设置”，确认公共地址、字段长度、数据点配置和突发上送周期。
5. 在“数据点管理”中添加或批量生成数据点。
6. 点击“启动”。
7. 告知外部主站连接本机 IP 和端口。
8. 外部主站连接后，在“连接列表”确认连接。
9. 在“报文监控”查看总召唤、控制命令及响应。
10. 测试过程中可在“数据点管理”修改点值，观察突发上送和对端显示。

## 9. 常见问题处理

### 9.1 客户端连接失败

请按以下顺序检查：

1. IP 地址是否正确。
2. 端口是否正确，默认 IEC104 端口为 `2404`。
3. 服务端是否已启动并正在监听。
4. Windows 防火墙或安全软件是否拦截。
5. 公共地址是否与对端一致。
6. COT、CA、IOA 字段长度是否与对端一致。
7. 对端是否限制客户端 IP 或最大连接数。

### 9.2 服务端启动失败

常见原因：

1. 端口已被占用。
2. 当前用户没有绑定该端口的权限。
3. 绑定地址不是本机有效地址。
4. 防火墙或安全软件阻止监听。

处理建议：

- 将端口改为 `2405`、`2406` 等未占用端口。
- 绑定地址改为 `0.0.0.0` 或 `127.0.0.1`。
- 关闭占用端口的其他程序。

### 9.3 有报文但数据页签不显示

可能原因：

1. 收到的 ASDU 类型不在当前分类支持范围内。
2. IOA 范围配置限制了显示范围。
3. 公共地址或字段长度与对端不一致，导致解析不符合预期。

处理建议：

- 在客户端“设置”中将对应数据类型的数量设为 `0`，表示不限制 IOA 范围。
- 查看“报文监控”中的报文详情，确认 ASDU 类型和 IOA。
- 与对端核对公共地址、COT 长度、CA 长度和 IOA 长度。

### 9.4 Excel 打开 CSV 乱码

工具导出的 CSV 已写入 UTF-8 BOM，正常情况下 Excel 可直接识别。

如仍显示异常：

1. 在 Excel 中选择“数据” -> “自文本/CSV”导入。
2. 文件原始格式选择 `UTF-8`。
3. 分隔符选择“逗号”。

### 9.5 2404 端口无法访问

IEC104 默认使用 TCP `2404` 端口。如果跨机器测试无法访问：

1. 确认服务端绑定地址不是 `127.0.0.1`。
2. 确认客户端访问的是服务端真实网卡 IP。
3. 检查 Windows 防火墙入站规则。
4. 检查两台机器是否在同一网络或路由可达。
5. 使用其他端口临时测试，排除端口限制。

## 10. 使用注意事项

1. 遥控、设点、复位等命令具有控制含义，仅建议在仿真、实验或明确授权的测试环境中使用。
2. 连接真实站端或生产系统前，应确认测试窗口、操作范围和安全措施。
3. 公共地址、字段长度、IOA 地址分区应与被测设备配置一致，否则可能出现连接成功但数据解析异常。
4. 保存的 `.jsonl` 报文文件适合回看和归档；导出的 `.csv` 文件适合统计、筛选和报告整理。
5. `.iec104` 场景文件用于保存客户端连接或服务端配置，便于复用测试环境。
